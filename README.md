# English

## Project Overview
One-Hour-Per-Game is an experimental project exploring what kind of web games can be created with AI-assisted development tools in **just one hour**. Each game is completed under a strict 60-minute time constraint, showcasing the potential and application of modern AI tools in game development.

## Project Goals
- Explore the boundaries of AI-assisted development in game creation
- Provide a Minimum Viable Product (MVP) example for each game type
- Document the application of AI technology in rapid game prototyping
- Share experiences and tips for using AI tools in game development
- Build an open-source repository showcasing implementations of different game types

## Game Plan
Consider adding different types of games in the future, such as:
- Simple 2D platform jumping games
- Puzzle games (matching, Sudoku, etc.)
- Card games
- Text adventure games
- Games with simple RPG elements
- Small shooting games

## Time Limit Rules
- Concept design, coding, and basic testing must be completed within 60 minutes
- Timing includes using AI tools to generate code, modify, and debug
- Games must be playable, even if simple or with minor flaws
- After time ends, only obvious errors can be fixed; no new features can be added

## Tech Stack
- Frontend: HTML5, CSS3, JavaScript/TypeScript
- Frameworks: React, Vue, Phaser.js, etc. depending on game requirements
- AI Tools: Claude, ChatGPT, GitHub Copilot, etc.
- Hosting: GitHub Pages

## How to Run
Each game is in its own folder with complete source code and running instructions. Typical steps:

```bash
# Clone the repository
git clone https://github.com/tobenot/One-Hour-Per-Game.git

# Enter a specific game directory
cd One-Hour-Per-Game/[game-name]

# Install dependencies (if any)
npm install

# Run the game
npm start
```

## 项目简介
One-Hour-Per-Game是一个实验性项目，旨在探索使用AI辅助开发工具在**仅仅一小时内**能创建出什么样的Web游戏。每个游戏都是在严格的60分钟时间限制下完成的，展示了现代AI工具在游戏开发中的潜力和应用。

## 项目目标
- 探索AI辅助开发在游戏创作中的能力边界
- 为每种游戏类型提供一个最小可行产品(MVP)示例
- 记录AI技术在快速游戏原型设计中的应用
- 分享使用AI工具进行游戏开发的经验和技巧
- 建立一个展示不同游戏类型实现的开源资源库

## 时间限制规则
- 每个游戏的概念设计、编码和基本测试必须在60分钟内完成
- 计时包括使用AI工具生成代码、修改和调试的时间
- 游戏必须是可玩的，即使功能简单或有小缺陷
- 在时间结束后，只允许修复明显的错误，不能添加新功能
